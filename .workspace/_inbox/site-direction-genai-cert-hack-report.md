## 1. サイトの一言定義

- **名称**: Syllabus Hack（`syllabushack.com`）
- **コアメッセージ**: **生成AIでシラバス（試験範囲）をハックする** — 暗記中心の旧来型学習に代わり、生成AIとデータを武器に **効率的かつ本質理解に近い** 資格学習を提案する日本語メディア兼ツール群。
- **SEO 上の自己紹介（要約）**: GenAI で資格試験を攻略する「シラバスハック」。ITパスポート・基本情報・セキュマネ等の効率学習と最新トレンドを発信（`src/config.yaml` の `metadata.description`）。

## 2. 理念（ナレッジ化の主軸）

### 2.1 問題意識

- 資格試験学習が **用語の丸暗記・量の押し付け** に偏りがちであることへの批判的スタンス。
- 一方で **試験というゲームのルール（シラバス・出題形式・配点）** は無視できない — そこを **「ハック」** する（最短ルート・捨て問・CBT操作・採点者視点など）実用主義。

### 2.2 解としての「生成AI × シラバス」

| 観点 | 内容 |
|------|------|
| **シラバス** | IPA 等の公式範囲を正として、追従・差分予測・用語の構造化に AI を使う |
| **生成AI** | 家庭教師・ドリル生成・図解（Mermaid 等）・長文要約・添削・記憶術（画像・比喩）の **学習オーケストレーター** |
| **投資対効果** | 合格だけでなく、プロンプト力・情報の構造化・ツール習熟が **実務に直結** する「一石二鳥」を明示（ガイド記事の「3つの柱」と整合） |

### 2.3 モットー（サイト内の定式）

- **「生成AI × シラバス ＝ 未来への投資」** — 試験対策と AI リテラシー・業務での AI 活用を同じレッスンに束ねる。

## 3. プロダクトの方向性（2軸）

トップページのコピーどおり、提供価値は **実践（CBT・アプリ）** と **効率（メソッド・ツール）** の二軸。

1. **CBT 設問アプリ / インタラクティブ**  
   スマホ対応、試験 UI に近い演習、分野別ドリル、AI 解説など。`src/apps/` の Preact 製クイズ・シミュレーターと連動。

2. **AI 学習メソッド**  
   NotebookLM、Gemini、ChatGPT を使った暗記・問題生成・概念可視化・論文・記述対策など、「AI を家庭教師にする」具体手順。

加えてトップでは **最新トレンド（IPA 改訂・CBT・シラバス追従）** と **用語解説（辞書・深掘り）** を第3・第4の柱として提示。

## 4. コンテンツカテゴリと役割

公式の5カテゴリ（`trend` / `method` / `career` / `app` / `theory`）に加え、実装上 **strategy（戦略・横断テーマ）**、**term / glossary（用語集）**、開発記 **app 記事** が厚く、全体像は次のとおり。

| 領域 | 方向性 |
|------|--------|
| **method** | プロンプト、NotebookLM、直前対策、捨て問、メンタル・睡眠、CBT 操作など **再現可能な手順** が中心 |
| **trend** | IPA の CBT 化・シラバス改訂・生成AI 出題・AI 倫理・高度区分の動向など **試験環境の変化** |
| **career** | 資格の市場価値、履歴書、手当交渉、地方・職種別の **資格の使い道** |
| **theory** | 経営戦略・ネットワーク・PM 等の **頻出理論**（theory は共通カバー画像で統一） |
| **app** | クイズ・シミュレーター紹介と **開発ストーリー**（透明性・メンテ方針） |
| **strategy / term** | ITパスポート横断の応用解説・用語辞典（シラバス語彙の定着） |

**編集上のトーン**: 受験者の不安（時間・不合格・会社都合）に寄り添いつつ、**ハック・逆転・最短** など行動を促す語彙。比喩が多く、読み物としてのエンタメ性も意識。

## 5. 対象試験・トピックの優先度

- **中核**: 情報処理技術者試験（ITパスポート → FE → AP → 高度区分、SG 等）。
- **周辺**: 生成AIパスポート・生成AI導入実務者検定、FP、AWS 学習ロードマップなど **IT リテラシー・AI 時代の資格**。
- **横断テーマ**: CBT 移行、2027 IPA 再編・データマネジメント区分など **制度変更への追随** が継続的テーマ。

## 6. 技術スタック（サイトとしての「実践」の証明）

- **静的サイト**: Astro 6 + AstroWind、日本語、`src/config.yaml` で SEO・GA。
- **アプリ**: `src/apps/` に Preact 製クイズ・CBT 風 UI、`src/data/master/` に問題 JSON。
- **メタ**: 記事は page bundle（`index.md` / `index.mdx` + `cover.jpg`）、フロントマターで `knowledge.examId` 等を付与する設計。

→ **「AI で記事もアプリも回す」** ことがコンテンツ（開発記・AI 駆動アーキテクチャ記事）でも語られる。

## 7. Claude ナレッジ利用時の注意（解釈ガイド）

1. **「ハック」は不正ではなく**、シラバス・試験設計・学習科学に沿った **戦略とツール活用** を指す。
2. **生成AIは万能ではない** — ハルシネーション対策、試験本番のルール、著作権・AI倫理は trend / method で繰り返し扱う **リスク意識** とセット。
3. **読者は主に日本語の受験者** — 制度名・略称（IPA、CBT、FE、AP 等）は日本の資格コンテキスト前提。
