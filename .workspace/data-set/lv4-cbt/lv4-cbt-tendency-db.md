高度試験（スキルレベル4）の2025年（ペーパー最後）〜2026年（CBT開始）の過渡期における出題傾向を分析し、シラバスVer 5.x（生成AI/DX追加）を反映したデータベースです。

## 1. 全区分共通：最重要トレンド（Core Trends）

| トレンド項目 | 内容・出題のポイント | キーワード |
| :--- | :--- | :--- |
| **DXの深化** | 単なるデジタル化から「ビジネス価値（PoV）」の創出へ。DSS（デジタルスキル標準）に準拠した論点。 | PoV, DSS, 人的資本経営, リスキリング |
| **生成AIの実装** | 「仕組み」の理解から「実務実装・ガバナンス」へ。RAGとファインチューニングの使い分けが頻出。 | RAG, LLM, ファインチューニング, ハルシネーション |

| **モダンセキュリティ** | 境界防御から「ゼロトラスト」へ。ソフトウェアサプライチェーンのリスク管理。 | ゼロトラスト, SASE, SBOM, NIST CSF 2.0 |

## 2. 試験区分別：CBT/最新出題ポイント（Exam Specifics）

### 【論文系：ST / SA / PM / AU】
AIを「魔法の杖」とせず、**「不確実性をどう管理するか」**がA評価の分かれ目。

- **ST（ストラテジスト）**: 経営層へのAI導入提案。ROI算出（コスト削減だけでなく売上増の二軸評価）とガバナンス体制の構築。
- **SA（アーキテクト）**: AI Agent（自律型AI）の設計。推論コスト（トークン課金）の最適化（ルーティング、非同期化、キャッシュ）。
- **PM（マネージャ）**: 100%の精度を保証できないAIプロジェクトの品質管理。「撤退基準」の合意とHITLによる品質担保。
- **AU（監査）**: AI事業者ガイドラインに基づくAIガバナンス監査。CAAT（コンピュータ支援監査技法）としての生成AI活用。

### 【記述・専門系：SC / NW / DB / ES / SM】
複雑な情報の「視覚化」「構造化」がCBT対策としても有効。

- **SC（セキュリティ）**: Security for AI（プロンプトインジェクション対策/ガードレール）とAI for Security（アノマリ型WAF/IDS）。
- **NW（ネットワーク）**: AIOpsによる自律運用。HTTP/3 (QUIC) や SASE によるクラウド通信の最適化。
- **DB（データベース）**: ベクトルDBとRAGの実装。ハイブリッド検索（ベクトル＋キーワード）とチャンク分割（チャンキング）の設計。
- **ES（エンベデッド）**: エッジAI（オンデバイスAI）の機能配分（クラウド vs エッジ）。安全性（フェールセーフ/フォールバック）とリアルタイム性のトレードオフ。
- **SM（サービスマネージャ）**: AIエージェントによる一次対応自動化とエスカレーション要件。確率的出力に対するSLA（サービスレベル合意）の再定義。

## 4. CBT移行期のハック戦略（Hack Strategy）

1. **「タイピング記述」への適応（2026年〜）**:
   - CBTでは「図を書く」設問が減り、「構造化されたテキスト（Markdown風や箇条書き）」での解答が求められる可能性が高い。
   - 論文対策では、手書きの「字の綺麗さ」から「論理構造の明快さ」へ評価の重みがシフトする。

2. **「複雑な状況の自動整理」スキルの重視**:
   - CBTの問題文は画面上で読むため、スクロールによる情報見落としのリスクが高い。
   - AIを使って「長文問題からタイムラインや構成図を自動生成する」練習（メソッド）が極めて有効になる。

3. **最新用語の定義の「不変性」**:
   - シラバスVer 5.xで追加されたAI用語は、どの区分でも午前IIの四択問題として「確実な得点源」になるため、全区分共通でマスターすべき。
