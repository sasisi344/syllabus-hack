# ドラフト品質チェック

対象ファイル: $ARGUMENTS

指定されたファイル（またはディレクトリ内の全ファイル）を品質チェックしてください。
引数が空の場合は `.workspace/draft/` 内の全ファイルを対象にする。

## チェック項目

### 1. フロントマター必須フィールド確認

以下がすべて存在するか確認する:

- [ ] `publishDate` — ISO形式（T00:00:00Z 付き）
- [ ] `lastmod` — YYYY-MM-DD 形式
- [ ] `title` — 30〜40文字推奨
- [ ] `excerpt` — 約100文字
- [ ] `image` — `~/data/post/{category}/{slug}/cover.jpg` 形式
- [ ] `category` — trend / method / career / app / theory のいずれか
- [ ] `tags` — 3〜5個
- [ ] `draft` フィールドの存在確認
- [ ] `metadata.description` — 120〜160文字
- [ ] `persona` フィールドの存在確認
- [ ] `knowledge.examId` フィールドの存在確認

### 2. Bold 変換チェック

```
Grep パターン: \*\*[^*]+\*\*
```

- `**bold**` 形式が残っている箇所をすべてリストアップ
- 本番移行前に `<strong>` への変換が必要な箇所を明示

### 3. 画像パス検証

- `image:` フィールドが `~/data/post/` から始まっているか
- 相対パス（`./` `../`）や絶対パス（`/src/`）が混入していないか
- IMAGE_PROMPT コメントの有無を確認

### 4. ヘッダー規約確認

- `「」` 括弧を使った見出しがないか
- `## 1. タイトル` のような番号付き見出しがないか

### 5. タグ正規化確認

以下のエイリアスが使われていないか確認する（tag_rules.md の正規タグ名を使用すること）:

- `ITパスポート` ✓ / `IP`, `Iパス` ✗
- `基本情報技術者` ✓ / `FE`, `基本情報` ✗
- `応用情報技術者` ✓ / `AP`, `応用情報` ✗
- `情報処理安全確保支援士` ✓ / `SC`, `登録セキスペ` ✗

## 出力フォーマット

各チェック項目について `✅ OK` / `⚠️ 要修正: {詳細}` の形式で報告する。
問題がある場合は修正コマンドを提案する（実行はユーザーの判断に委ねる）。
