<h2>試験での出題ポイント</h2>
<p>試験では、OODAを構成する4つのステップの意味と、PDCAとの適した場面の違いが問われます。</p>
<ol>
<li><strong>Observe（観察）</strong>: 生のデータや現場の状況をありのままに見る。</li>
<li><strong>Orient（方向づけ）</strong>: 自分の経験や知識を使い、今の状況が何を意味するか解釈する。（※ここが最も重要）</li>
<li><strong>Decide（決定）</strong>: 具体的に何をするか決める。</li>
<li><strong>Act（実行）</strong>: 動いてみて、その結果をまた「観察」に戻す。</li>
</ol>
<h3>PDCAとの決定的な違い</h3>
<ul>
<li><strong>PDCA</strong>: 「計画（Plan）」から始まる。既存業務の効率化や品質管理に向いている。</li>
<li><strong>OODA</strong>: 「観察（Observe）」から始まる。想定外の出来事が起きる新規事業や戦場のような現場に向いている。</li>
</ul>
