## 3行まとめ
*   <strong>UI (User Interface)</strong>: ユーザーが製品やサービスと接する「窓口」となる部分（ボタン、文字、画面構成など）。
*   <strong>UX (User Experience)</strong>: 製品やサービスを通じてユーザーが得る「体験（楽しさ、満足感、使い心地など）」全体を指す。
*   <strong>関係性</strong>: 優れた UI は優れた UX を生むための「手段」の1つです。

## 試験での出題ポイント
試験では、特に「UI」と「UX」の区別と、その「目的」が問われます。

1.  <strong>UI</strong>:
    -   例：サイトのレイアウト、フォント、アイコンの形、クリックの感触。
    -   ポイント： <strong>接触する表面</strong> という「見た目や操作性」を指します。
2.  <strong>UX</strong>:
    -   例：アプリを使って目的が果たせた満足感、注文から配送までのスムーズさ。
    -   ポイント： <strong>一連の流れによる体験</strong> という「感情や価値」を指します。
3.  <strong>HCD (Human Centered Design)</strong>: 人間中心設計。企画の初期段階から理想の UX を追求し、ユーザーの意見を反映させる設計プロセスのことです。

## 【AIハック】生成AIで「満足度の源」を探る
UIとUXのロジックは、AIに「アプリ改善のリサーチ」をシミュレートさせることで理解を深められます。

### プロンプト例:
> 「あなたはアプリ開発のディレクターです。ある旅行予約アプリについて『画面はきれい（ <strong>UI</strong> ）だけど、予約が完了するまでが多くて疲れる（ <strong>UX</strong> ）』という不満がありました。 <strong>人間中心設計 (HCD)</strong> の考え方を使って、 <strong>UX (体験)</strong> を向上させるための具体的な改善案を、試験用語を交えて3つ提案してください。」

AIが「プロセスの簡略化」や「ユーザーテスト（行動観察）」といった、具体的な改善ステップを提示してくれるため、体験向上のためのデザイン思考が身に付きます。

## まとめ：「形」を超えた「心地よさ」を作ろう
UIは「見せる」もの、UXは「感じさせる」ものです。

試験では、 <strong>外観や操作</strong> という言葉があれば UI、 <strong>満足感や体験</strong> という言葉があれば UX と結びつけて考えましょう。
