## 3行まとめ
* <strong>プロンプトエンジニアリング</strong>は、生成AIから目的に合った回答を得るために、指示文（プロンプト）を設計する技術。
* 重要なのは「質問のうまさ」ではなく、<strong>目的・文脈・制約・出力形式</strong>を明示して再現性を高めること。
* ITパスポートでは、生成AI活用の実務知識として、品質向上とリスク低減の両面で問われる。

## 試験での出題ポイント
試験では、次の観点が狙われやすいです。

1. <strong>役割指定</strong>: 「あなたは情報セキュリティ担当者です」のように前提を固定する。  
2. <strong>文脈付与</strong>: 誰向け・何の目的かを示して、回答の粒度をそろえる。  
3. <strong>制約設定</strong>: 文字数、禁止事項、出力形式（箇条書き/表）を指定する。  
4. <strong>検証指示</strong>: 根拠確認、曖昧点の列挙、再チェック観点を含める。

## なぜ重要か（実務での価値）
同じAIでも、プロンプトの設計次第で回答品質は大きく変わります。  
特に業務利用では、情報漏えい、誤回答、過剰一般化を防ぐために、最初から制約付きで指示することが重要です。

たとえば、社内文書の要約を依頼する場合でも、  
- 個人情報を伏せて要約する  
- 不確実な内容は「推定」と明記する  
- 断定表現を避ける  
といった条件を入れることで、リスクを抑えながら活用できます。

## 基本テンプレート（そのまま使える形）
以下の型を使うと、初心者でも品質を安定させやすくなります。

1. <strong>目的</strong>: 何のための出力か  
2. <strong>対象読者</strong>: 誰が読むか  
3. <strong>入力データ</strong>: 何を材料にするか  
4. <strong>制約条件</strong>: 文字数、禁止事項、トーン  
5. <strong>出力形式</strong>: 箇条書き、見出し、チェックリスト  
6. <strong>検証観点</strong>: 誤りや不足の自己点検

## オプトアウトポリシー記事とのつながり
オプトアウトポリシーを運用する現場では、プロンプト設計もガバナンス対象です。  
なぜなら、入力内容そのものが機密情報や個人情報を含むリスクがあるためです。

そのため、次の2点をセットで運用するのが実務的です。
- <strong>ポリシー</strong>: 何を入力してよいか（データ分類ルール）  
- <strong>プロンプト設計</strong>: どの形式でAIに指示するか（再現性と検証性）

## 【AIハック】最短で身につける練習法
### プロンプト例
> 「あなたは社内のAI活用ルール策定担当です。『議事録を要約して上司に報告する』業務について、機密情報を漏らさず、誤解を生みにくいプロンプトを3つ作成してください。各プロンプトに、目的・制約・検証観点を1行ずつ添えてください。」

## まとめ
プロンプトエンジニアリングは、AIを賢く使うためのテクニックであると同時に、品質管理とリスク管理の基礎技術です。  
試験対策では定義と要素を押さえ、実務では「再現性のある指示テンプレート」を持つことが成果につながります。
