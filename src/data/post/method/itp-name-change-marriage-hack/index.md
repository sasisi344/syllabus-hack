試験の申込完了後に入籍した。

気づいたとき、画面には「申込者氏名：旧姓〇〇」の文字。マイナンバーカードには「新姓〇〇」。

IPAのサポートに電話すると、「氏名変更は可能ですが、窓口での手続きが必要です。試験まで日が迫っている場合、間に合わない可能性があります」という回答が返ってくる。

<strong>公式の「仕様です」で終わる場所に、Syllabus Hack は踏み込む。</strong>

---

## これがなぜ「詰み」になるのか

CBT方式のIPA試験は、試験当日に本人確認書類の提示が必要だ。会場スタッフは「申込名と証明書の氏名が一致していること」を確認する。

不一致があった場合の対応は会場によって異なるが、「原則として申込名と一致が必要」という立場を取られると、その場で受験拒否になる可能性がある。

そして受験料は<strong>原則返金されない</strong>。

試験センターへの変更手続きには数日以上かかることが多く、直前の申し込みや急な結婚では物理的に間に合わない。

## 「第3の解決策」という発想

IPAが示す正規ルートは2つだ。

1. 変更手続きを完了させる（間に合わない場合がある）
2. 受験を諦めて再申込する（お金と時間が消える）

だが実際には<strong>第3の選択肢がある</strong>。「氏名が変更になった事実と経緯を証明する文書」を試験会場に持参し、事前にスタッフへ状況を説明するという方法だ。

これは公式の手続きではないが、会場スタッフが「本人確認が取れた」と判断できるだけの証拠を揃えることで、受験を認めてもらえる可能性が高まる。

## 用意する3点セット

以下を試験当日に持参する。

1. **旧姓での申込確認メール**（IPAから届いた受験票）
2. **新旧氏名が確認できる書類**（旧姓が記載された運転免許証 + 新姓のマイナンバーカード、あるいは戸籍謄本・抄本）
3. **AIで作成した「氏名変更説明文書」**（下記参照）

3番目が今回の核心だ。

## AIで作る「氏名変更説明文書」プロンプト

以下のプロンプトをそのまま ChatGPT・Claude・Gemini に貼り付けて実行する。

```
以下の状況について、試験センターのスタッフに提示する「氏名変更に関する説明文書」を日本語で作成してください。

【状況】
- 試験名：ITパスポート試験（または〇〇試験）
- 試験日：〇〇年〇〇月〇〇日
- 申込時の氏名（旧姓）：〇〇 〇〇
- 現在の氏名（新姓）：〇〇 〇〇
- 改姓理由：令和〇年〇月〇日の婚姻による氏名変更
- 持参書類：マイナンバーカード（新姓）、旧姓記載の運転免許証

【文書要件】
- A4用紙1枚に収まる分量
- 本人が作成する誓約書・説明書の形式
- 「申込者本人と受験者が同一人物であること」を明確に証明する内容
- 丁寧なビジネス文体
- 署名欄を末尾に設ける
```

出力された文書を印刷し、実際に署名を入れて持参する。

## 試験当日の動き方

会場に到着したら、受付でスタッフに<strong>先に状況を説明する</strong>。後手に回ると「原則どおり」の判断をされやすくなる。

「申込後に婚姻で氏名が変わり、変更手続きが間に合いませんでした。旧姓と新姓両方が確認できる書類と説明文書を持参しています」と伝え、3点セットを提示する。

多くの会場では、これで受験を認めてもらえる。ただし会場によって対応が異なるため、<strong>試験前日までに試験センターに電話で相談しておく</strong>ことが最も確実だ。

## AIハックポイント

説明文書の作成だけでなく、試験センターへの問い合わせ電話のスクリプトもAIに作らせることができる。

```
「ITパスポート試験の申込後に改姓しました。試験当日に旧姓での申込と新姓の本人確認書類が一致しない状態になります。会場での対応について確認したいのですが、どのような手順を踏めばよいでしょうか？」
という電話をする際の、落ち着いて話せる台本を作ってください。
```

公式サポートは「正規手続き」しか教えてくれない。<strong>AIは「あなたが今置かれた状況で動ける選択肢」を整理してくれる。</strong>

---

## まとめ

改姓後の氏名不一致は、放置すると受験拒否という最悪の結末を生む。だが「公式手続きが間に合わない」＝「詰み」ではない。

AIで作る説明文書 + 旧新氏名の証明書類 + 事前の電話確認という3ステップで、この問題は乗り越えられる。

試験当日に後悔しないために、今夜のうちにAIを起動しよう。
